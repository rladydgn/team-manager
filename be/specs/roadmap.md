# 백엔드 로드맵

각 단계는 작고 검증 가능한 단위로 진행한다. 단계가 바뀌면 명세,
구현 계획, 검증 문서를 함께 갱신한다.

## Phase 1 - 현재 기능 정리

- 기존 사용자, 팀, 경기 API 목록 정리
- 공통 응답과 에러 형식 문서화
- 현재 도메인 규칙과 누락된 테스트 파악

## Phase 2 - 팀 관리 안정화

- 팀 생성/조회/상세 조회 계약 확정
- 팀원 역할과 상태 규칙 정리
- 팀 권한 검증 테스트 보강

## Phase 3 - 경기 관리 안정화

- 외부 경기와 내부 경기 생성 규칙 확정
- 경기 상태 전이와 참가자 규칙 정리
- 경기 API 검증 테스트 보강

## Phase 4 - 멤버십과 회비

- 회비 도메인 개념 정의
- 납부 상태, 기간, 알림 규칙 명세
- 관련 API와 데이터 모델 설계

## Phase 5 - 운영 준비

- 로그와 에러 관측 기준 정리
- 운영 설정 분리
- 배포 전 회귀 검증 절차 정리
