# 기능 요구사항

## 기능 이름

회원가입 계정 식별자 고유성

## 목표

회원가입의 아이디와 이메일을 애플리케이션 및 데이터베이스에서 모두 고유하게 보장한다.

## 인수 기준

- 아이디와 이메일은 필수 입력이며 공백 값은 허용하지 않는다.
- 사용 중인 아이디는 `DUPLICATED_USERNAME`으로 거부한다.
- 사용 중인 이메일은 `DUPLICATED_EMAIL`으로 거부한다.
- `users.username`과 `users.email`은 각각 DB 고유 키를 가진다.
- 아이디·이메일 중복 확인 API가 사용 가능 여부를 반환한다.
