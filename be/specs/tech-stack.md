# 백엔드 기술 스택

## 현재 선택

| 영역 | 선택 | 기준 |
| --- | --- | --- |
| 언어 | Kotlin | 타입 안정성과 간결한 도메인 표현 |
| 빌드 | Gradle Kotlin DSL | Kotlin/Spring 프로젝트 표준 구성 |
| 프레임워크 | Spring Boot | 웹 API, 설정, 테스트 생태계 |
| 데이터 접근 | Exposed 기반 테이블/엔티티 구조 | 현재 코드베이스의 패턴 유지 |
| 인증 | JWT | 프론트엔드와 분리된 토큰 기반 인증 |
| API 문서 | OpenAPI 설정 | 클라이언트 계약 확인 |
| 테스트 | Spring/Kotlin 테스트 | 서비스, 컨트롤러, 통합 검증 |

## 설계 기준

- 컨트롤러는 HTTP 입출력과 인증 컨텍스트 처리에 집중한다.
- 서비스는 유스케이스와 도메인 규칙을 표현한다.
- Repository는 데이터 접근과 조회 조건을 캡슐화한다.
- DTO는 API 계약을 표현하며 도메인 객체를 직접 노출하지 않는다.
- 공통 응답과 에러 응답은 일관된 형식을 유지한다.

## 아직 선택하지 않은 것

- 운영 DB와 마이그레이션 도구
- 배포 방식
- 관측 도구
- 고도화된 권한 모델
