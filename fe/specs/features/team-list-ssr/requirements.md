# 팀 목록 초기 렌더링

## 배경

`/teams`는 브라우저가 로드된 뒤 팀 목록을 요청해 첫 화면에 목록 대신 로딩
상태가 노출된다.

## 목표

- `/teams`의 최초 팀 목록을 서버에서 조회해 첫 HTML에 포함한다.
- 검색, 팀 생성, 가입 신청, 재시도 동작은 기존 클라이언트 상호작용을 유지한다.

## 제외 범위

- 팀 상세와 경기 일정 화면의 SSR 전환
- 팀 목록 API 계약 변경

## 사용자 이야기

- 방문자로서, 팀 목록 페이지를 열면 팀 목록을 바로 확인할 수 있어야 한다.

## 인수 기준

- Given 팀 목록 API가 성공하면, When `/teams`를 처음 요청하면, Then 팀 목록 또는 빈 상태가 첫 서버 렌더링에 표시된다.
- Given 팀 목록 API가 실패하면, When `/teams`를 처음 요청하면, Then 오류 안내와 재시도 버튼이 표시된다.
- Given 팀 생성 또는 재시도 후, When 목록을 다시 조회하면, Then 최신 팀 목록으로 갱신된다.

## 화면 상태

| 상태 | 설명 |
| --- | --- |
| Success | 서버에서 조회한 팀 카드 또는 빈 상태를 표시한다. |
| Error | 목록 오류를 표시하고 클라이언트 재시도를 제공한다. |
| Loading | 생성 후 또는 재시도 중에만 기존 로딩 상태를 표시한다. |

## API 계약

| 항목 | 내용 |
| --- | --- |
| Endpoint | `/teams` |
| Method | `GET` |
| Request | 없음 |
| Response | `CommonResponse<Team[]>` |
| Error | 서버 오류 시 초기 오류 UI |
