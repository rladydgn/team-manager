# 검증

## 자동화 검증

| 검증 | 명령 | 기대 결과 |
| --- | --- | --- |
| Lint | `npm run lint` | 통과 |
| Build | `npm run build` | 통과 |

## 수동 화면 검증

1. 등록된 팀이 있는 상태에서 `/teams`에 직접 진입해 팀 카드가 첫 화면에 표시되는지 확인한다.
2. 팀이 없는 상태에서 빈 상태와 생성 폼이 표시되는지 확인한다.
3. 백엔드를 중지한 뒤 오류 상태와 재시도 버튼이 표시되는지 확인한다.
4. 팀을 생성한 뒤 최신 목록으로 갱신되는지 확인한다.

## 인수 기준 매핑

| 인수 기준 | 검증 방법 | 결과 |
| --- | --- | --- |
| 최초 목록 서버 렌더링 | production build 및 수동 검증 | build 통과, 수동 확인 대기 |
| 오류와 재시도 표시 | 수동 검증 | 대기 |
| 생성 후 갱신 | 수동 검증 | 대기 |
