# 구현 기록

## 구현 상태

- 상태: 구현 완료

## 변경 사항

- `sign-up-validation.ts`에 백엔드와 동일한 이름, 아이디, 비밀번호 검증 규칙을 추가했다.
- 회원가입 페이지에 필드별 오류 문구, 빨간 테두리, 접근성 속성을 추가했다.
- `/users/id/check` 호출과 `DUPLICATED_USERNAME` 오류 처리를 추가했다.
- HTTP 클라이언트가 API 오류 코드와 상태를 보존하도록 `ApiRequestError`를 추가했다.
- 이메일을 필수 입력으로 전환하고, 프론트와 백엔드에서 공백 값을 거부하도록 했다.
