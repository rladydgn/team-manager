# 기능 요구사항

## 기능 이름

회원가입 프론트엔드 검증

## 목표

회원가입 요청 전에 프론트엔드가 백엔드와 동일한 입력 규칙을 검사하고, 실패한 필드를 명확히 안내한다.

## 사용자 시나리오

- 사용자가 필드에서 포커스를 벗어나면 해당 값의 형식을 검사한다.
- 사용자가 가입을 요청하면 이름, 아이디, 비밀번호를 모두 검사한다.
- 형식이 올바른 아이디는 중복 확인 API로 사용 가능 여부를 확인한다.
- 실패한 입력은 빨간 테두리와 경고 문구로 표시한다.

## 인수 기준

- 이름은 공백 제거 후 비어 있지 않고 50자 이하여야 한다.
- 아이디는 영문 소문자, 숫자, `-`, `_`만 사용한 5~20자여야 한다.
- 비밀번호는 영문, 숫자, 특수문자를 모두 포함한 8~20자여야 한다.
- 중복 아이디와 가입 API의 중복 응답은 아이디 필드 오류로 표시한다.
- 이메일은 공백 제거 후 비어 있지 않은 필수 입력이어야 한다.
