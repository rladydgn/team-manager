# 기능 요구사항

## 기능 이름

회원가입 프론트엔드 검증

## 목표

회원가입 요청 전에 백엔드와 같은 입력 규칙 및 아이디·이메일 중복 여부를 확인하고, 실패한 필드를 명확히 안내한다.

## 인수 기준

- 이름, 아이디, 비밀번호, 이메일의 형식과 필수 여부를 검사한다.
- 아이디와 이메일은 중복 확인 API로 사용 가능 여부를 확인한다.
- 중복 아이디 또는 이메일은 해당 필드의 빨간 테두리와 경고 문구로 표시한다.
- 가입 요청의 `DUPLICATED_USERNAME`, `DUPLICATED_EMAIL` 응답도 해당 필드 오류로 표시한다.
