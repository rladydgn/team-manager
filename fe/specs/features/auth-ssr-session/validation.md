# 검증

## 자동화 검증

| 검증 | 명령 | 기대 결과 |
| --- | --- | --- |
| Lint | `npm run lint` | 통과 |
| Build | `npm run build` | 통과 |

## 수동 화면 검증

1. 유효한 액세스 쿠키로 `/`에 직접 진입해 헤더와 메인 화면이 로그인 상태로 바로 표시되는지 확인한다.
2. 쿠키 없이 `/`에 직접 진입해 로그인/회원가입 UI가 표시되는지 확인한다.
3. 액세스 쿠키를 만료시키고 유효한 리프레시 쿠키를 유지한 뒤, 클라이언트의 기존 세션 복구가 동작하는지 확인한다.

## 인수 기준 매핑

| 인수 기준 | 검증 방법 | 결과 |
| --- | --- | --- |
| 유효한 액세스 쿠키의 첫 로그인 UI | 수동 검증 | 대기 |
| 비인증 요청의 안전한 UI | 수동 검증 | 대기 |
| lint와 build 통과 | 자동화 검증 | 통과 |
