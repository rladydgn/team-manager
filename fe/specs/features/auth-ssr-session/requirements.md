# 인증 초기 렌더링

## 배경

유효한 로그인 상태에서도 클라이언트가 리프레시 요청을 끝낼 때까지 비로그인
UI가 먼저 노출된다.

## 목표

- 유효한 액세스 토큰 쿠키가 있으면 첫 서버 렌더링부터 로그인 사용자 UI를 표시한다.
- 인증 토큰은 브라우저 JavaScript에 노출하지 않는다.

## 제외 범위

- 서버 측 리프레시 토큰 재발급
- 세션 데이터베이스 또는 BFF 프록시 도입

## 사용자 이야기

- 로그인 사용자로서, 첫 화면을 열 때 비로그인 화면이 잠시 보이지 않아야 한다.

## 인수 기준

- Given 유효한 액세스 쿠키, When 페이지를 처음 요청하면, Then 로그인 사용자 UI가 첫 HTML에 포함된다.
- Given 액세스 쿠키가 없거나 유효하지 않으면, When 페이지를 요청하면, Then 비로그인 UI가 안전하게 표시된다.
- Given 액세스 쿠키가 없고 리프레시 쿠키가 유효하면, When 클라이언트가 준비되면, Then 기존 재발급 흐름으로 로그인 상태를 복구한다.

## 화면 상태

| 상태 | 설명 |
| --- | --- |
| Success | 서버가 현재 사용자를 전달해 로그인 UI를 바로 표시한다. |
| Unauthorized | 비로그인 UI를 표시하고, 클라이언트에서 기존 리프레시 흐름을 시도한다. |
| Error | 백엔드 연결 실패 시 비로그인 UI를 안전하게 표시한다. |

## API 계약

| 항목 | 내용 |
| --- | --- |
| Endpoint | `/users/me` |
| Method | `GET` |
| Request | `team_manager_access_token` HttpOnly 쿠키 |
| Response | `CommonResponse<UserLoginResponse>` |
| Error | 인증 실패 시 `401` |
