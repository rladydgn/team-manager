# 프론트엔드 기술 스택

## 현재 선택

| 영역 | 선택 | 기준 |
| --- | --- | --- |
| 언어 | TypeScript | UI 상태와 API 계약의 타입 안정성 |
| 프레임워크 | Next.js App Router | 현재 코드베이스 구조 유지 |
| UI | React | 선언적 화면 구성 |
| 스타일 | CSS / 글로벌 스타일 | 현재 프로젝트의 단순한 스타일링 유지 |
| API 통신 | shared api/http helper | 인증과 에러 처리 일관성 |
| 인증 상태 | features/auth model | 현재 인증 세션 패턴 유지 |
| 검증 | ESLint, 수동 UI 검증 | 기본 품질 확인 |

## 설계 기준

- `src/app`은 라우트와 페이지 조립에 집중한다.
- `src/features`는 도메인별 API와 화면 로직을 담는다.
- `src/shared`는 공통 설정, API 클라이언트, 인증 토큰 같은 기반 코드를
  담는다.
- 화면은 로딩, 빈 상태, 실패 상태, 성공 상태를 명시적으로 표현한다.
- API 에러는 사용자에게 이해 가능한 메시지로 변환한다.

## 아직 선택하지 않은 것

- 컴포넌트 라이브러리
- E2E 테스트 도구
- 서버 상태 캐싱 도구
- 디자인 토큰 체계
