# 프론트엔드 로드맵

각 단계는 작고 검증 가능한 사용자 흐름으로 진행한다. 단계가 바뀌면
명세, 구현 계획, 검증 문서를 함께 갱신한다.

## Phase 1 - 현재 화면 정리

- 로그인, 회원가입, 팀 목록, 팀 상세, 경기 화면 흐름 정리
- 현재 API 연동과 에러 처리 방식 문서화
- 모바일/데스크톱에서 깨지는 화면 여부 확인

## Phase 2 - 팀 관리 경험 안정화

- 팀 생성/조회/상세 화면의 상태 정의
- 팀원 역할과 권한에 따른 UI 노출 규칙 정리
- API 실패와 빈 상태 UX 보강

## Phase 3 - 경기 관리 경험 안정화

- 경기 목록, 상세, 생성 흐름 명세
- 외부 경기와 내부 경기 입력 차이 정리
- 참가자 상태 표시와 검증 보강

## Phase 4 - 멤버십과 회비 화면

- 회비 목록, 납부 상태, 기간 필터 설계
- 운영자와 팀원별 화면 차이 정의
- 백엔드 계약에 맞춘 API 연동

## Phase 5 - 제품 품질 보강

- 접근성 점검
- 반응형 레이아웃 정리
- 핵심 플로우 회귀 검증 절차 정리
